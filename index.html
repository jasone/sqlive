Overview

SQLIVE is a general-purpose SQL database that lets you subscribe to changes. Through a new construction called "virtual views", one can ask the database to maintain a view up-to-date at all times, get notified when the view changed, or produce a "diff" between now and any time in the past.

Streaming

SQLIVE can also process ephemeral streams of data. Those can be used to receive alerts or compute real-time analytics. Of course it is possible to mix those ephemeral streams with SQL tables, through joins or other constructions.

Concurrent

SQLIVE supports a robust subset of SQL (including transactions) but is also highly concurrent. Multiple readers/writers are supported and complex queries can be run without stalling the other users of the database.

Built with Skip

SQLIVE was entirely written in a proprietary fork of the language Skip (http://skiplang.com). Skip is a programming language originally developed by Facebook (open-sourced in 2018). Skip's unique features, such as the tracking of side effects to provide caching with reactive invalidation, ergonomic and safe parallelism, and efficient garbage collection made it the perfect choice to build SQLIVE.
